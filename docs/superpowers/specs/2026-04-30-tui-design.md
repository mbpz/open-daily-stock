# daily_stock_analysis TUI 设计文档

**日期:** 2026-04-30
**项目:** daily_stock_analysis → TUI 改造
**状态:** 设计完成，待用户确认

---

## 1. 概述与目标

将 `daily_stock_analysis` 项目改造成支持 TUI（终端用户界面）的应用。TUI 作为用户与系统的主要交互界面，提供分层导航式操作体验。

**设计原则：**
- YAGNI — 仅实现明确需求，不做 speculative 功能
- 简洁性 — 每个模块专注于单一职责
- 可维护性 — 使用 Textual 框架，代码清晰可扩展

---

```
│  ┌─────────────────────────────────────┐ │
│  │                                     │ │
│  │         Main Content Area           │ │
│  │   (Markets / Tasks / Analyze /      │ │
│  │    Config / Logs  五个子视图)         │ │
│  │                                     │ │
│  └─────────────────────────────────────┘ │
│  ┌─────────────────────────────────────┐ │
│  │           Footer Bar                │ │
│  │  状态栏 | 通知提示 | 最后更新时间      │ │
│  └─────────────────────────────────────┘ │

```

### 2.2 数据流向（混合式）

```
                    ┌──────────────────┐
                    │   TUI Frontend   │
                    │   (Textual App)  │
                    └────────┬─────────┘
                             │
              ┌──────────────┼──────────────┐
              │              │              │
              ▼              ▼              ▼
     ┌────────────┐  ┌────────────┐  ┌────────────┐
     │  行情模块  │  │ 任务/配置  │  │  日志模块  │
     │ (独立获取) │  │ (复用Pipeline)│ │ (子进程)  │
     └────────────┘  └────────────┘  └────────────┘
              │              │              │
              ▼              ▼              ▼
     ┌─────────────────────────────────────────┐
     │          data_provider + analyzer       │
     │    (akshare / tushare / baostock)       │
     └─────────────────────────────────────────┘
```

**说明：**
- **行情数据** — TUI 内置 DataProviderWrapper，独立调用数据源，自动轮询
- **任务/分析/配置** — 复用 `StockAnalysisPipeline` 和 `Config`，与 main.py 一致
- **日志** — 通过 `subprocess` 启动 main.py，实时显示完整输出

### 2.3 模块划分

| 模块 | 职责 | 数据来源 |
|------|------|----------|
| Markets | 自选股实时行情，自动轮询 | data_provider (独立) |
| Tasks | 分析任务进度、历史记录 | Pipeline (复用) |
| Analyze | 手动输入代码触发分析 | Pipeline (复用) |
| Config | 自选股列表、数据源开关 | Config (复用) |
| Logs | 实时日志输出 | subprocess pipe |

## 5. 功能详情

### 5.1 行情模块（Markets）

**数据获取：**
- 启动时加载自选股列表
- 使用 `akshare` 获取 A 股/港股数据，`yfinance` 获取美股数据
- 并发获取，失败时显示上次有效价格 + 状态标记

**刷新机制：**
- 自动轮询间隔：默认 30 秒（可配置 10s/30s/60s）
- 手动刷新：`r` 键
- 后台静默刷新，不阻塞 UI

**行情异动检测：**
- 涨跌超 5% 时在状态栏提示
- 可配置异动阈值

### 5.2 任务模块（Tasks）

**任务来源：**
- 当前运行中的分析任务（通过 Pipeline 事件）
- 历史任务记录（从 `reports/` 目录读取）

**状态展示：**
- `🔄` 进行中（带进度百分比）
- `✅` 已完成
- `❌` 失败

### 5.3 分析模块（Analyze）

**工作流：**
1. 输入股票代码（支持单码和多码，逗号分隔）
2. 按 Enter 开始分析
3. 调用 `StockAnalysisPipeline`
4. 实时显示分析进度（子进程 stdout）
5. 分析完成后显示结果摘要

**结果展示：**
- 简要结论 + 买卖点位
- 完整报告可查看/复制

### 5.4 配置模块（Config）

**配置项：**
- 自选股列表（可编辑）
- 数据源开关（AkShare/Tushake/Baostock/YFinance）
- 通知渠道开关
- 刷新间隔设置

**持久化：**
- 修改后直接写入 `.env` 文件
- 备份原文件为 `.env.backup`

### 5.5 日志模块（Logs）

**日志来源：**
- 启动 main.py 子进程，实时接收 stdout/stderr
- TUI 自身操作日志

**过滤功能：**
- 按级别过滤：all/info/warning/error
- 搜索功能（模糊匹配）

## 6. 技术实现

### 6.1 技术栈

| 组件 | 技术 | 说明 |
|------|------|------|
| 框架 | Textual | Python TUI 框架 |
| 数据源 | AkShare / YFinance | 通过现有 data_provider |
| 配置 | 复用 src.config | 不引入新配置层 |
| 分析 | 复用 StockAnalysisPipeline | 与 main.py 复用逻辑 |
| 日志 | subprocess + pipe | 接收 main.py 输出 |

### 6.2 项目结构

```
daily_stock_analysis/
├── tui/
│   ├── __init__.py
│   ├── main.py              # TUI 入口
│   ├── app.py               # Textual App 主类
│   ├── widgets/
│   │   ├── __init__.py
│   │   ├── header.py        # 顶部状态栏
│   │   ├── footer.py       # 底部状态栏
│   │   ├── markets.py      # 行情模块
│   │   ├── tasks.py        # 任务模块
│   │   ├── analyze.py      # 分析模块
│   │   ├── config.py       # 配置模块
│   │   └── logs.py         # 日志模块
│   ├── data/
│   │   └── wrapper.py       # 数据层封装
│   └── styles/
│       └── theme.py        # 主题样式
└── ...
```

### 6.3 核心组件

**App 类：**
- 管理全局状态（current_module, stocks_list, tasks）
- 处理全局快捷键
- 协调子模块通信

**各 Module 类：**
- 继承 `Textual.Widget`
- 各自管理渲染和交互
- 通过 `app` 访问全局状态

**DataWrapper 类：**
- 封装 data_provider 调用
- 管理轮询定时器
- 提供统一的行情数据接口

## 7. 错误处理

### 7.1 数据获取失败
- 显示 `---` 占位 + 红色 `❌` 标记
- 工具提示显示错误原因
- 不阻塞其他股票显示

### 7.2 分析任务失败
- 任务状态标记为 `❌`
- 错误信息展示在详情中
- 不影响其他任务继续

### 7.3 子进程异常退出
- 日志模块显示错误信息
- 状态栏提示连接断开
- 提供重启子进程选项

---

## 8. 非目标（Out of Scope）

- 不支持 WebUI 与 TUI 同时运行（互斥）
- 不实现完整的回测功能
- 不支持复杂的图表渲染（仅数值展示）
- 不实现远程服务器管理

---

## 9. 实现顺序

1. **基础框架** — Textual App + 布局 + 导航
2. **行情模块** — 数据展示 + 轮询
3. **任务模块** — 任务列表
4. **分析模块** — 触发分析 + 结果展示
5. **配置模块** — 配置编辑
6. **日志模块** — 子进程 + 实时输出
7. **集成测试** — 全流程验证

## 10. 备选方案

### 方案 B：Blessed 框架

**优点：** 轻量、成熟、与现有代码风格接近

**缺点：**
- 布局需要手动计算位置
- 动画支持较弱
- 需要额外处理键盘事件

**结论：** Textual 的声明式布局和内置组件更适合分层导航架构。

### 方案 C：命令面板模式

**替代设计：**
```
┌─────────────────────────────────────────┐
│  > 命令面板                              │
├─────────────────────────────────────────┤
│  :analyze 600519                        │
│  :config                               │
│  :refresh                              │
│  :quit                                 │
└─────────────────────────────────────────┘
```

**结论：** 命令面板更适合高级用户，但降低了入门友好度。选择分层导航作为默认交互模式。
