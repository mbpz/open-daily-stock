# open-daily-stock 首次启动引导设计

## 目标

帮助小白用户快速完成初始配置，开始使用。

## 问题

当前首次启动没有任何引导，用户面对空白 Markets 界面不知道该做什么。

## 解决方案

### 引导触发条件

满足以下任一条件，显示首次启动引导：
- .env 文件不存在
- AI API Key 未配置
- 自选股列表为空

### 引导流程（3 步，分步表单）

| 步骤 | 内容 | 可跳过 |
|------|------|--------|
| 1 | API Key 配置（OpenAI/MiniMax/Gemini） | 否 |
| 2 | 自选股列表 | 否 |
| 3 | 通知渠道（可选） | 是（跳过按钮） |

### 交互方式

- **导航**：↑↓ 选择字段，Enter 下一项，Esc 跳过当前步骤
- **进度显示**：底部显示 "步骤 X/3"
- **跳过机制**：第 3 步有"跳过"按钮，跳过后稍后可去 Config 完整配置

### 跳过引导后

- 进入 Markets 模块
- 顶部显示提示条："⚠️ 请先配置（按 4 进入 Config）"
- 5s 后提示条自动消失，但保留在 Config 模块的高亮提示

### 引导完成后

直接进入 Markets 模块，开始显示行情数据。

## 技术实现

### 新增文件

- `tui/widgets/wizard.py` — 引导页面组件

### 修改文件

- `tui/app.py` — 检测引导条件，优先显示引导页
- `src/config.py` — 读取/保存配置到 .env

### 状态定义

```python
# Config 单例增加方法
def is_first_time_setup() -> bool:
    """检查是否需要显示首次启动引导"""
    return not Path('.env').exists() or not self.openai_api_key or not self.stock_list

# TUIApp 状态
self._show_wizard = config.is_first_time_setup()
```
